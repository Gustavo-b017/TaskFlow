## 🔍 1. Descobertas da Auditoria "Bug-Hunting"

Após uma revisão minuciosa (double-check) com foco em falhas, as seguintes melhorias foram implementadas:

### A. Robustez na Restauração de Sessão (`AuthContext.tsx`)
* **Falha Detectada:** O `JSON.parse` no carregamento inicial do `AsyncStorage` não possuía tratamento de erro. Dados corrompidos no storage local poderiam causar o crash do app no boot.
* **Correção:** Implementado bloco `try/catch` aninhado. Se o parse falhar, o sistema agora limpa a chave corrompida e permite que o app siga para a tela de login com segurança.
* **Impacto:** Eliminação de crash crítico no ciclo de vida inicial.

### B. Consistência de Estado e UI (`LoginScreen.tsx`)
* **Audit de Redundância:** Verificado o uso de estados de `loading`. O sistema agora sincroniza corretamente o feedback visual, garantindo que o botão de login fique desabilitado e mostre o spinner durante todo o processo assíncrono, protegendo contra múltiplas submissões (Race Conditions).
* **Feedback ao Usuário:** Validado o uso de `Alert.alert` para erros de credenciais, garantindo que o usuário saiba exatamente por que a autenticação falhou.

### C. Redirecionamento por Role (`AppRoutes.tsx` - FIAP-0024)
* **Verificação de Regra:** Confirmado que o `initialRouteName` respeita estritamente a role do usuário:
    * `admin` -> Redireciona para aba `Settings`.
    * `user` -> Redireciona para aba `Home`.
* **Segurança:** A lógica de roteamento é reativa ao estado do `AuthContext`, impedindo acesso a rotas privadas antes do login (RootNavigator).

## 🛡️ 2. Auditoria de Segurança (CVE & CVE Bugs)

* **Vulnerabilidades Detectadas (npm audit):**
    * `uuid < 14.0.0`: Moderate (Missing buffer bounds check).
    * `@tootallnate/once < 3.0.1`: Moderate (Incorrect Control Flow Scoping).
* **Análise de Risco:** Ambas as vulnerabilidades estão em dependências transitórias do `expo` e `jest-expo`. Não há exposição direta via código de domínio do TaskFlow.
* **Mitigação:** Documentado o risco. A atualização para versões seguras exige `npm audit fix --force`, o que resultaria em breaking changes no framework Expo. Recomendado agendamento de migração de versão do Expo para o próximo ciclo.
