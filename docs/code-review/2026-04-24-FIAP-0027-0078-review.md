## 🔍 1. Auditoria de Contratos TypeScript (FIAP-0027)

### Vulnerabilidades Corrigidas
* **Segurança de Identidade (`user.ts`):** 
    * **Falha:** O campo `password` estava exposto na interface de domínio `User`.
    * **Risco:** Vazamento inadvertido de credenciais em logs de estado ou componentes de UI.
    * **Correção:** Campo removido do modelo de domínio. Criada a interface local `AuthUserSource` no `AuthContext` para isolar a senha apenas no momento da autenticação.
* **Integridade de Tipagem:** 
    * Validada a ausência de `any`. Uso estrito de union types para `TaskStatus`, `TaskPriority` e `UserRole`.

## 🔍 2. Auditoria de Persistência de Tratamento (FIAP-0078)

### Refatorações de Robustez
* **Sanitização de Input (`TreatmentService.ts`):**
    * **Falha:** O serviço confiava cegamente no valor retornado pelo `AsyncStorage`.
    * **Correção:** Implementada validação via `VALID_TREATMENTS.includes()`. Agora, se o storage local for corrompido ou injetado com valores inválidos (Object Injection risk), o sistema ignora o valor e retorna `null`, garantindo estabilidade.
* **Desacoplamento:** 
    * A lógica de persistência foi totalmente movida para um Service, permitindo que o hook `useTreatment` permaneça focado apenas no estado reativo.
