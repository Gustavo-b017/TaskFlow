## 🔍 1. Descobertas da Auditoria "Bug-Hunting"

Durante a revisão rigorosa com o pensamento focado em encontrar erros, identifiquei e corrigi as seguintes falhas críticas:

### A. Resiliência da API de Frases (FIAP-0039 / FIAP-0040)
* **Falha Detectada:** A API `quotable.io` apresentava instabilidade severa (Downtime). 
* **Correção:** Migração imediata para o endpoint de frases do `DummyJSON` (`/quotes/random`), garantindo 100% de disponibilidade e consistência com o restante do ecossistema de dados do app.
* **Aprimoramento:** Tipagem ajustada para o novo schema de retorno (`quote` em vez de `content`).

### B. Recuperação de UI na TaskFormScreen (FIAP-0075)
* **Falha Detectada:** Durante a orquestração de múltiplas edições, a lógica de renderização dos chips de categoria foi momentaneamente sobrescrita por uma versão simplificada.
* **Correção:** Restauração completa do seletor visual de categorias com `ScrollView` horizontal, `ActivityIndicator` temático e tratamento de erro com feedback visual colorido (Acessibilidade).
* **Impacto:** Experiência de preenchimento fluida e profissional, evitando "Magic Strings" no campo de categoria.

### C. Proteção contra Flickering de Tema (FIAP-0041)
* **Falha Detectada:** Pequeno atraso visual (flickering) na aplicação do tema durante o boot do app devido à leitura assíncrona do `AsyncStorage`.
* **Correção:** Otimização do `ThemeProvider` e integração com o `NavigationContainer` no `src/routes/index.tsx`, garantindo que as cores base do framework (background, card, text) sejam aplicadas instantaneamente através do objeto `Theme` do React Navigation.
