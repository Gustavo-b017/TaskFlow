## 🔑 2. Sistema de Autenticação (FIAP-0064, 0065, 0066)

### A. AuthContext e useAuth
* **Refatoração de Segurança:** O `AuthContext` foi blindado para nunca expor senhas no estado global do app. A lógica de `signIn` filtra o objeto `User` antes de salvá-lo.
* **Resiliência do Hook:** O `useAuth` inclui uma verificação explícita de existência do contexto, disparando um erro descritivo em tempo de desenvolvimento se o `AuthProvider` for esquecido.

### B. LoginScreen
* **UX e Performance:** 
    * Implementado `KeyboardAvoidingView` para garantir usabilidade em telas menores.
    * Feedback visual de loading no botão via `ActivityIndicator`.
    * Validação síncrona de campos antes de disparar chamadas assíncronas.
* **Segurança:** Bloqueio de múltiplas submissões simultâneas via estado de `loading`.
